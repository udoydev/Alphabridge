# Alphabridge - Professional Consulting Services Platform

A modern Django-based website showcasing consulting, AI, and digital transformation services.

## Features

- **Service Showcase** - Display 5 main service categories with detailed sub-services
- **Service Details** - Individual pages for each service category
- **Professional Design** - Modern, responsive UI built with Bootstrap 5
- **Contact Page** - Simple contact information card
- **Home Page** - Hero section with service slider and CTA buttons

## Tech Stack

- **Backend**: Django 4.2.8
- **Frontend**: Bootstrap 5, FontAwesome
- **Server**: Gunicorn
- **Static Files**: WhiteNoise
- **Deployment**: Vercel

## Local Development Setup

### Prerequisites
- Python 3.11+
- pip

### Installation

1. Clone the repository
```bash
git clone <repository-url>
cd Alphabridge
```

2. Create virtual environment
```bash
python -m venv .venv
source .venv/bin/activate  # On Windows: .venv\Scripts\activate
```

3. Install dependencies
```bash
pip install -r requirements.txt
```

4. Create .env file (copy from .env.example)
```bash
cp .env.example .env
```

5. Update .env with your settings
```
SECRET_KEY=your-secure-secret-key-here
DEBUG=True  # For development only
ALLOWED_HOSTS=localhost,127.0.0.1
```

6. Run migrations
```bash
python manage.py migrate
```

7. Collect static files
```bash
python manage.py collectstatic --noinput
```

8. Run development server
```bash
python manage.py runserver
```

Visit http://localhost:8000

## Production Deployment on Vercel

### Prerequisites
- Vercel Account (https://vercel.com)
- GitHub Account
- Git installed locally

### Step 1: Initialize Git Repository
```bash
git init
git add .
git commit -m "Initial commit"
```

### Step 2: Create GitHub Private Repository
1. Go to https://github.com/new
2. Name: `Alphabridge`
3. Select **Private**
4. Click "Create repository"
5. Follow GitHub's instructions to push your code

```bash
git remote add origin https://github.com/YOUR_USERNAME/Alphabridge.git
git branch -M main
git push -u origin main
```

### Step 3: Deploy to Vercel
1. Visit https://vercel.com/dashboard
2. Click "Add New Project"
3. Import your GitHub repository
4. Configure settings:
   - Framework: Other (Django)
   - Root Directory: ./
   - Build Command: `pip install -r requirements.txt && python manage.py collectstatic --noinput`
   - Output Directory: `staticfiles`

### Step 4: Set Environment Variables in Vercel
In Vercel Dashboard:
1. Go to Settings > Environment Variables
2. Add the following:

```
SECRET_KEY = [Generate a secure key from Django]
DEBUG = False
ALLOWED_HOSTS = your-vercel-domain.vercel.app,*.vercel.app
SECURE_SSL_REDIRECT = True
SESSION_COOKIE_SECURE = True
CSRF_COOKIE_SECURE = True
```

### Step 5: Deploy
Click "Deploy" and wait for the deployment to complete.

## Environment Variables

Required environment variables for production:

| Variable | Description | Example |
|----------|-------------|---------|
| SECRET_KEY | Django secret key | Generated by Django |
| DEBUG | Debug mode | False |
| ALLOWED_HOSTS | Allowed hosts | your-domain.com,*.vercel.app |
| SECURE_SSL_REDIRECT | Force HTTPS | True |
| SESSION_COOKIE_SECURE | Secure cookies | True |
| CSRF_COOKIE_SECURE | Secure CSRF | True |

## Static Files

Static files are served using WhiteNoise for Vercel compatibility. Run this before deployment:

```bash
python manage.py collectstatic --noinput
```

## Database

Currently configured for SQLite (suitable for small projects). For production with high traffic:
- Consider PostgreSQL
- Update DATABASES in settings.py
- Set DATABASE_URL environment variable

## Troubleshooting

### Static files not loading on Vercel
```bash
python manage.py collectstatic --noinput --clear
```

### 500 errors on Vercel
1. Check Vercel logs in dashboard
2. Verify environment variables are set correctly
3. Ensure ALLOWED_HOSTS includes your Vercel domain

### DEBUG mode in production
Always set `DEBUG=False` in production environment variables.
